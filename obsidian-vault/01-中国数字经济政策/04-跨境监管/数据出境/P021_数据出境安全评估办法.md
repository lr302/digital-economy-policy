# 数据出境安全评估办法

> **立法背景**: 《网络安全法》第37条（CIIO数据本地化）、《数据安全法》第21/25条（重要数据+出口管制）、《个人信息保护法》第38-40条（个人信息出境三路径）均设定了数据出境的前置条件，但均缺少操作细则 → 2022年7月国家互联网信息办公室令第9号公布 → 2022年9月1日施行 → 2024年3月《促进和规范数据跨境流动规定》(P077)发布后，部分评估触发条件放宽、特定场景免评

## 核心摘要

> **一句话概括**：确立数据出境安全评估为最高门槛的法定合规路径，以六种强制申报情形划定适用范围，构建"自评估→申报→受理→组织评估→结论→复议"六段程序，评估重点涵盖出境合法性/正当性/必要性、数据出境风险、合同法律约束力、接收方所在国法律环境四大维度，评估结论有效期2年，是三部上位法数据出境条款的核心实施细则。

**制定背景**：

数据跨境流动已成为数字经济的核心议题。全球范围内，欧盟2018年实施GDPR确立了"充分性认定+标准合同条款(SCC)"的双轨出境机制，美国则倡导"信任的数据自由流动"(Data Free Flow with Trust, DFFT)理念，主张降低数据本地化壁垒。2021年，全球已有140多个国家制定了数据保护立法，各国在数据主权与数据自由流动之间的博弈日趋激烈。

中国在2017年《网络安全法》第37条首次写入"关键信息基础设施运营者(CIIO)境内存储+出境安全评估"要求，但此后五年一直缺乏操作层面的实施细则。2021年《数据安全法》《个人信息保护法》相继出台，分别确立了重要数据出境安全评估制度和个人信息出境的三条合规路径（安全评估、标准合同、认证），但"谁来评、怎么评、评什么"的问题始终悬而未决。

在此背景下，网信办于2022年7月发布本《办法》，填补了三法的程序性空白。这是中国数据出境制度的第一块"硬砖"——确立了安全评估作为最高门槛路径的地位。此后，2023年2月标准合同办法(P024)发布，2024年3月促进规范数据跨境流动规定(P077)发布，三大出境路径全部成形，并开始逐步放宽部分场景的评估要求，体现了从"全面严控"到"精准监管"的政策演进逻辑。

## 核心条款精读

### 一、强制申报的六种情形（第4条）🔑

本法最核心的条款。以下任一情形，数据处理者须通过网信办安全评估方可向境外提供数据：

| 序号 | 情形 | 制度逻辑 | P077(2024)的调整 |
|------|------|---------|-----------------|
| 1 | 向境外提供**重要数据** | 维护国家安全——重要数据涉及国家核心利益 | 未放宽。重要数据的认定需依据各地区/行业的重要数据目录 |
| 2 | **CIIO**向境外提供个人信息 | 维护关键信息基础设施安全——CIIO掌握的数据量大、敏感度高 | 未放宽。CIIO认定依赖关键信息基础设施保护条例 |
| 3 | 处理**100万人以上**个人信息的数据处理者向境外提供个人信息 | 规模门槛——大型平台数据出境的社会影响面广 | **P077放宽**：非重要数据+非CIIO的百万级处理者，若出境场景属于P077免评范围可免评 |
| 4 | 自上年1月1日起累计向境外提供**10万人以上**个人信息 | 累计量门槛——防止"拆包"规避 | P077提高了部分量化门槛的具体认定标准 |
| 5 | 自上年1月1日起累计向境外提供**1万人以上**敏感个人信息 | 敏感信息特别保护——敏感信息的泄露风险更高 | 敏感个人信息的范围参照《个保法》第28条 |
| 6 | 国家网信部门规定的其他情形 | 兜底条款——保留监管弹性 | — |

**深度解析——"100万人"门槛的制度设计逻辑**：

"处理100万人以上个人信息"这一门槛是整部办法最具争议和影响的条款。其制度设计逻辑在于：
- **覆盖超级平台**：微信（月活13亿）、抖音（日活7亿）、淘宝（年活9亿）等超级App的数据处理者，其任何个人信息出境行为都将触发安全评估
- **不以实际出境量为准**：这与第4条情形4、5的"累计出境量"门槛不同——只要处理者处理的个人信息总量超过100万人，即使仅向境外传输1条数据，也需评估
- **实务影响巨大**：几乎所有中型以上互联网企业均落入此范围，导致2022-2024年间安全评估申请量远超预期

**教学点**：这个"100万"门槛是固定值还是浮动标准？立法时为何选择100万而非50万或200万？这反映了对监管资源与企业合规成本的权衡——门槛太低则审批工作量过大，太高则漏管风险增加。2024年P077的放宽说明监管层意识到原标准可能"用力过猛"。

### 二、申报材料与自评估（第5-6条）

**事前的数据出境风险自评估**是申报安全评估的前置条件（第5条），体现了"企业自查+政府复核"的双层审查模式。

**自评估重点**（第5条）：
1. **数据出境目的、范围、方式的合法性、正当性、必要性** — 需论证"为何必须出境"，不能是"便于管理"等泛化理由
2. **出境数据的规模、范围、种类、敏感程度** — 需量化并分级
3. **境外接收方的数据安全保障能力** — 需评估接收方的技术措施、管理制度
4. **数据出境对国家安全、公共利益、个人权益的风险** — 需做风险评估矩阵
5. **与境外接收方拟订立的合同**是否充分约定了数据保护义务 — 合同中需包含PIPL第38条要求的条款
6. **其他**可能影响数据出境安全的事项

**申报书及材料**（第6条）：
- 申报书（网信办提供模板）
- 数据出境风险自评估报告（核心材料，工作量最大）
- 数据处理者与境外接收方拟订立的合同或其他法律文件
- 安全评估工作需要的其他材料（如数据出境的具体业务场景说明、技术架构图等）

**实务提示**：自评估报告的编制难度是大多数企业申报过程中的主要瓶颈。企业通常需要：法务团队梳理出境数据流、安全团队评估接收方保护能力、外部律师出具法律意见书、技术团队绘制数据流图。一份完善的自评估报告动辄数十页。

### 三、评估程序与时限（第7-14条）🔑

数据出境安全评估的程序流程可以用下图概括：

```
数据出境需求
    ↓
【步骤1】事前自评估（第5条）
    ↓（编制自评估报告+签订合同）
【步骤2】向网信办提交申报材料（第6条）
    ↓
【步骤3】网信办受理审查——7个工作日内决定是否受理（第7条）
    ├── 材料不齐全 → 退回并告知补正
    └── 材料齐全 → 书面受理通知
         ↓
【步骤4】网信办组织安全评估（第10条）
    ↓（45个工作日内完成）
    ├── 情况复杂 → 可适当延长（告知申报人）
    ↓
【步骤5】作出评估结论（第12条）
    ├── 准予出境 → 有效期2年（第13条）
    ├── 不准出境 → 书面说明理由
    └── 附条件准予出境
         ↓
【步骤6】（不服结论时）申请复议（第14条）
```

**关键时限节点**：

| 节点 | 法定时限 | 实务提示 |
|------|---------|---------|
| 受理审查 | 7个工作日 | 材料补正时间不计入 |
| 组织评估 | 45个工作日 | 可延长，延长次数无硬性上限（但需告知） |
| 评估结论有效期 | 2年 | 到期需重新申报 |
| 已开展出境活动整改期 | 6个月（第15条） | 自办法施行之日(2022.9.1)起算 |

**评估结论的法律性质**：安全评估的结论实质上是一种"行政许可"，申报人需在评估结论有效期内严格按照申报内容进行数据出境活动。若数据出境的目的、方式、范围、种类发生变化，需重新申报（第9条）。

### 四、评估重点事项（第8条）

国家网信部门在安全评估中重点评估以下事项：

| 评估维度 | 具体内容 | 评估逻辑 |
|---------|---------|---------|
| **合法性** | 数据出境是否符合法律、行政法规规定 | 形式审查：是否有法律依据 |
| **正当性** | 数据出境的目的、方式是否正当 | 实质审查：目的是否合理、方式是否适当 |
| **必要性** | 数据出境范围是否为实现目的所必需 | 最小必要原则：不许多传 |
| **出境风险** | 数据出境对国家安全、公共利益、个人权益的风险 | 影响评估：风险等级判定 |
| **合同保障** | 境外接收方承担的责任义务是否有法律约束力 | 合同审查：救济措施是否充分 |
| **法律环境** | 境外接收方所在国家/地区的法律环境和网络安全环境 | 国别评估：参照GDPR充分性认定逻辑 |
| **其他** | 其他可能影响数据出境安全的事项 | 兜底条款 |

**核心教学点**：第8条第(六)项"境外接收方所在国家或地区的法律环境和网络安全环境"的评估，实质上是中国版的"充分性认定"。与GDPR由欧盟委员会统一作出"充分性认定"不同，中国采取个案式"国别评估"模式——每个申报案件都要论证接收方所在国的法律环境是否充分保护数据安全，这大大增加了评估的复杂性和不确定性。实践中，向数据保护法律较完善的国家（如欧盟成员国、日本、韩国、新加坡等）传输数据的通过率显著更高。

### 五、重新评估情形（第9条）

在评估结论有效期内，出现以下情形之一，需**重新申报**安全评估：

1. 数据出境目的、方式、范围、种类发生变化，或延长境外保存期限
2. 境外接收方所在国家/地区法律环境发生变化，可能影响数据安全
3. 境外接收方的数据安全保障能力发生变化
4. 数据处理者或境外接收方发生重大变更（如实际控制权变更）
5. 国家网信部门规定需要重新评估的其他情形

**制度逻辑**：重新评估制度确保了评估结论的"动态有效性"——数据出境不是"一评定终身"。这与欧盟SCC制度中的"重新评估义务"类似，但中国政府的主动介入程度更高。

### 六、2024年政策调整（P077的关系）

2024年3月22日，《促进和规范数据跨境流动规定》(P077)颁布并施行，对安全评估制度作出了重要调整：

**放宽措施（P077的核心调整）**：

| 调整内容 | 原规则(P021) | P077新规则 |
|---------|------------|-----------|
| 非重要数据+特定场景 | 无豁免 | 国际贸易、跨境运输、学术合作、跨境生产制造等场景中非重要数据出境免评 |
| 自贸区负面清单 | 无 | 上海等自贸区可制定数据出境负面清单，清单外数据免评 |
| 员工个人信息 | 同等对待 | 按照依法制定的劳动规章制度和集体合同实施人力资源管理所必需的个人信息出境免评 |
| 合同必需场景 | 同等对待 | 订立、履行个人作为一方当事人的合同所必需的个人信息出境免评（如跨境购物、机票酒店预订） |
| 紧急情况 | 同等对待 | 紧急情况下为保护自然人的生命健康和财产安全所必需的个人信息出境免评 |
| 非重要数据+非CIIO+年出境不满10万人（不含敏感信息） | 需评估（若满足第4条其他情形） | 免予安全评估 |

**P077的"负面清单"制度创新**：
- 上海自贸区率先试点"数据出境负面清单管理规则"
- 负面清单明确列明需要安全评估的数据类型
- 清单外的数据出境免除安全评估程序
- 这是从"正面审查"到"负面清单管理"的制度转型，旨在降低合规成本

**教学点**：从2022年P021到2024年P077，中国数据出境制度经历了从"全面严控"到"精准监管"的转变。这一转变的驱动因素包括：①企业合规负担过重——安全评估申请积压严重，45个工作日的法定时限在实际中往往被延长；②对标国际——G20、APEC等场合强调促进数据自由流动；③经济需求——跨境电商、国际SaaS服务等场景对数据自由流动的刚性需求。

## 三大出境路径对比

| 维度 | 安全评估(P021) | 标准合同(P024) | 保护认证 |
|------|---------|---------------|------|
| 适用条件 | 六种法定情形（第4条） | 非评估情形的一般个人信息出境 | 关联公司间的个人信息出境 |
| 法律依据 | 《个保法》第38条第1款第1项 + 第40条 | 《个保法》第38条第1款第3项 | 《个保法》第38条第1款第2项 |
| 主管机关 | 国家网信办（省级初审） | 省级网信办备案 | 经网信办认可的专业机构 |
| 企业义务 | 事前自评估→提交申报→等待审批 | 签订标准合同→备案 | 通过认证→持续合规 |
| 审查方式 | 实质性安全评估 | 形式备案（事后监管） | 专业机构认证 |
| 审批周期 | 45个工作日+可延长 | 备案后即可生效 | 认证周期视机构而定 |
| 有效期 | 2年 | 合同有效期（可续签） | 认证周期（需年检） |
| 监管强度 | ★★★★★ 最高 | ★★★ 中等 | ★★ 较低（自我规制为主） |
| 违规后果 | 最重（参照三法最高罚款） | 较重 | 较重 |
| 申请人定位 | 特定规模/类型的数据处理者 | 中小型数据处理者 | 跨国集团内部传输 |

**路径选择的决策逻辑**：
1. 先判断是否落入第4条六种情形 → 若是，必须走安全评估
2. 若非安全评估情形 → 可在标准合同与认证之间选择
3. 跨国公司内部传输 → 认证路径可能更高效
4. 2024年P077后 → 先判断是否属于免评场景，再考虑路径选择

## 制度创新

| 创新点 | 内容 | 影响 |
|--------|------|------|
| **双层审查模式** | "企业自评估+政府安全评估"的双层机制。企业需先完成详尽的内部风险自评估，政府在此基础上进行独立审查。这与GDPR"数据保护影响评估(DPIA)"的思路一致，但增加了强制性的政府复核环节 | 企业合规成本大幅增加，但安全审查质量更高；自评估的独立性不足是实务痛点 |
| **量化触发门槛** | "100万人以上""10万人以上""1万人以上敏感个人信息"等量化标准，与国际上常见的"规模以上企业"或"高风险处理活动"等定性标准形成对比 | 量化标准更明确、可操作性强，但也导致"一刀切"——大量中型企业被动进入评估程序 |
| **个案式国别评估** | 对境外接收方所在国法律环境进行个案评估，而非欧盟式的统一"充分性认定"清单 | 增加了评估的不确定性，但也为中国保留了对不同国家的差异化政策空间 |
| **动态重新评估** | 出境条件变化时需重新申报，确保评估结论的动态有效性 | 增加了企业的持续合规成本，但避免了"一评定终身"的监管盲区 |
| **三路径分层设计** | 安全评估（高门槛）、标准合同（中门槛）、认证（低门槛）的三层结构化路径设计 | 实现了"分类分级"的数据出境管理，为不同类型和规模的企业提供了差异化合规方案 |

## 文件间关系

- **与《网络安全法》第37条**：CIIO的"数据本地化+出境安全评估"义务是本《办法》的上位法依据。《办法》第4条第2项将CIIO出境个人信息纳入强制评估范围，落实了《网安法》的原则性要求。
- **与《数据安全法》第21、25条**：第21条确立了重要数据目录制度（各地区/行业分别制定），第25条要求重要数据处理者进行数据出境安全评估。《办法》第4条第1项据此将"向境外提供重要数据"列入强制评估情形。《办法》第19条同时明确了"重要数据"的定义。
- **与《个人信息保护法》第38-40条**：PIPL第38条确立了个人信息出境的三条路径（安全评估、标准合同、认证），第39条规定了出境时的单独同意+告知义务，第40条规定了CIIO和处理100万人以上信息者的强制安全评估义务。本《办法》全面细化了这些条款的实施程序。
- **与P024（标准合同办法）**：并行路径。P024于2023年2月发布，2023年6月施行。凡不落入P021第4条六种情形的个人信息出境，可通过签订标准合同并备案的方式合规出境。
- **与P077（促进数据跨境流动规定）**：2024年3月发布，是对前两年严格管控的"纠偏"——免除特定场景（合同必需、跨境贸易、学术合作等）的安全评估要求，在自贸区试点负面清单管理，明确非重要数据+非CIIO+年出境不足10万人可免评。
- **与《促进和规范数据跨境流动规定》(P077)**：政策演变关系。P077的出台标志着中国数据出境监管从"严格审批"向"精准监管+促进流动"的阶段转变。

## 教学要点

### 一、课堂讨论

> "中国数据出境制度三年三变（2022安全评估→2023标准合同→2024放宽/负面清单），这反映了一种什么样的治理逻辑？"

**讨论引导**：
- **试错式监管**：先建立严格的顶层框架(P021)，在实践中发现合规成本过高、审批积压严重等问题后，再通过补充性立法(P024)和调整性立法(P077)逐步优化
- **安全与发展的动态平衡**：2022年偏重"安全"，2024年向"发展"倾斜——体现了中国数字经济治理中"统筹发展和安全"的核心原则
- **国际博弈中的制度弹性**：面对GDPR、CPTPP(要求数据自由流动)、DEPA等多边规则的压力，中国需要展示开放姿态，同时保留政策调控空间
- **制度学习与自主创新**：借鉴欧盟SCC、充分性认定等制度工具，但进行了本土化改造（如量化触发门槛、个案式国别评估）

### 二、国际比较

| 制度要素 | 中国(P021) | 欧盟(GDPR) | 美国 |
|---------|-----------|-----------|------|
| 法律基础 | 三部上位法+部门规章 | GDPR第5章(第44-49条) | 无统一联邦立法（CLOUD Act + 行业法） |
| 出境路径 | 安全评估/标准合同/认证三路径 | 充分性认定/SCC/BCR/认证等多路径 | 以"信任的自由流动"为主，无"充分性"机制 |
| 审查模式 | 事前审批制（安全评估路径） | 事后问责制（充分性认定除外） | 行业自律+事后执法 |
| 政府角色 | 深度介入（个案审批） | 框架设定+事后监管 | 轻监管（行业自律优先） |
| 对企业的影响 | 合规成本高，但制度明确 | 合规成本中等，制度成熟 | 合规成本低，但法律不确定性高 |
| CLOUD Act维度 | 数据本地化倾向（第37条） | 支持自由流动但需充分保护 | CLOUD Act允许跨境执法获取数据 |

**欧盟GDPR充分性认定制度（第45条）**：
- 欧盟委员会统一评估第三国的数据保护水平
- 获得"充分性认定"的国家/地区可自由接收欧盟数据（日本、韩国、英国等14个已获认定）
- 中国尚未获得欧盟充分性认定 → 中欧之间的数据传输主要通过SCC机制
- P021的个案式国别评估 vs GDPR的清单式充分性认定，是两种不同的制度路径

**美国CLOUD Act与跨境数据获取**：
- 2018年《澄清合法使用境外数据法》(CLOUD Act)：允许美国执法机构通过法律程序获取美国公司存储在海外的数据
- 同时允许符合"隐私和公民自由标准"的外国政府通过双边协议直接向美国科技公司请求数据
- 与中国的数据本地化要求形成制度冲突——中国要求数据存储于境内，美国要求美国公司"有就要给"

### 三、核心概念

| 概念 | 定义 | 出处 |
|------|------|------|
| **重要数据** | 一旦泄露可能危害国家安全、经济运行、社会稳定、公共健康和安全的数据 | 《数据安全法》第21条；《办法》第19条：各地区/行业制定重要数据目录 |
| **关键信息基础设施运营者(CIIO)** | 公共通信、能源、交通、水利、金融、公共服务、电子政务等重要行业的信息设施运营者 | 《网络安全法》第31条；《关键信息基础设施安全保护条例》 |
| **充分性认定(Adequacy Decision)** | 欧盟委员会对第三国数据保护水平的全面评估，认定通过后数据可自由流向该国 | GDPR第45条 |
| **标准合同条款(SCC)** | 欧盟委员会预先批准的个人数据跨境传输合同模板，企业签订后即满足GDPR出境要求 | GDPR第46条 |
| **数据本地化(Data Localization)** | 要求将数据存储在境内的法律义务，是数据出境的前置限制条件 | 《网络安全法》第37条 |
| **数据自由流动(Data Free Flow)** | 减少或消除数据跨境流动限制的理念，以促进数字贸易和全球经济一体化 | DFFT（日本G20倡议）；CPTPP第14章 |
| **负面清单管理** | 明确列出需要安全评估的数据类型，清单外的数据出境免评 | P077在自贸区试点 |

### 四、学生思考题

1. **"100万人以上"门槛**：某公司为一家30万用户的企业提供SaaS服务，该企业自身不直接处理个人信息，但SaaS平台存储了用户的个人信息（总量超过100万）。该SaaS公司是否需要安全评估？为什么？——（提示：判断"处理"的定义，参照PIPL第4条，区分"处理者"与"受托处理者"）

2. **国别评估的可操作性**：P021第8条第(六)项要求评估境外接收方所在国的"法律环境"。如果数据要传输到数据保护法律尚不完善的发展中国家（如部分非洲、东南亚国家），企业应如何论证"充分保护"？这种制度安排是否构成一种"数据贸易壁垒"？

3. **三路径的选择困境**：一个跨国零售企业在中国运营，需要将消费者购物数据传回欧洲总部用于供应链管理。该企业在华服务约200万注册用户，每年约5万用户的购买记录需传至海外。应选择哪条出境路径？如果路径选择错误，可能面临什么法律后果？

4. **P077的"松绑"逻辑**：2024年P077的放宽措施是否意味着中国放弃了数据主权的主张？在"安全"与"发展"之间，2022年的P021和2024年的P077分别侧重什么？这种政策摇摆对跨国企业的合规战略规划有何影响？

5. **向GPT等生成式AI提供数据**：当中国用户向OpenAI、Google Gemini等境外大模型输入问题时，是否构成"数据出境"？如果用户在提问中包含了个人信息（如要求翻译一封含有个人信息的邮件），应适用P021的哪一项触发条件？生成式AI时代的"数据出境"定义是否需要重新审视？
