# 企业数据资源相关会计处理暂行规定

> **出台背景**: 数据二十条提出"探索数据资产入表" → 财政部2023年8月出台会计规定 → 2024年1月施行

## 核心摘要

> **一句话概括**: 首次明确数据资源可确认为无形资产或存货列入资产负债表，标志着数据从"政策概念"正式跃升为可计量、可报告的会计要素，是"数据要素化"从理论走向制度落地的关键里程碑。

**制定背景**：

长期以来，企业积累的海量数据资源在传统会计框架下无法被确认为资产。企业为采集、清洗、加工数据所投入的巨额成本，只能作为当期费用化处理，导致资产负债表无法真实反映企业的数据资产价值，利润表被巨额费用侵蚀。这种"投入不计、价值不显"的困境，严重制约了企业数据资产化的积极性，也使得数据要素的市场化定价缺乏会计基础。

在国际层面，国际会计准则理事会（IASB）和美国财务会计准则委员会（FASB）均未出台专门针对数据资源的会计准则。数据资产在现行国际准则下只能零散地、间接地被部分条款所覆盖（如作为无形资产或商誉的组成部分），缺乏系统性的确认、计量和披露规范。这一空白使得全球企业在数据资产核算上面临同样的制度困境。

中国在这一领域率先破题。2022年12月"数据二十条"（《关于构建数据基础制度更好发挥数据要素作用的意见》）明确提出"探索数据资产入表新模式"，为会计制度的创新提供了顶层政策依据。财政部随后于2023年8月1日发布《企业数据资源相关会计处理暂行规定》（财会〔2023〕11号），自2024年1月1日起施行，走在了全球数据资产会计规范的最前沿。

## 全文结构

《暂行规定》全文结构清晰，分为总则性说明和两大类具体会计处理规范，形成"一个总则 + 两条路径 + 披露要求"的框架：

| 部分 | 核心内容 |
|------|----------|
| 总体要求 | 明确适用范围（执行企业会计准则的企业）、基本原则（遵循现有准则框架）、数据资源的定义与分类 |
| 数据资源作为无形资产的会计处理 | 确认条件、初始计量（外购/自行开发）、后续计量（摊销与减值）、处置与终止确认 |
| 数据资源作为存货的会计处理 | 确认条件、初始计量（采购成本/加工成本）、后续计量（成本与可变现净值孰低）、发出与结转 |
| 列示要求 | 资产负债表中的列示位置、报表项目名称的增设或分拆 |
| 披露要求 | 强制性披露内容（报表附注）与自愿性披露内容 |
| 附则 | 施行日期（2024年1月1日）、衔接规定 |

## 核心条款精读

### 一、数据资源入表的两条路径

本规定的核心设计在于：并非为数据资源创设全新的资产类别，而是将其"嵌入"现有企业会计准则的无形资产和存货框架中。这一设计既保持了会计准则体系的内在一致性，又实现了数据资源的资产确认。两条路径分别对应不同的业务模式：

**路径一：确认为"无形资产"**

适用于企业持有数据资源用于内部使用、对外提供服务或出租等长期使用场景。确认须同时满足以下条件：

1. **合法拥有或控制**：企业对数据资源享有合法的财产权利或实际控制力（这是最核心、也是实务中最具挑战性的条件）
2. **经济利益很可能流入**：与该数据资源有关的经济利益很可能流入企业
3. **成本能够可靠计量**：取得或开发该数据资源的成本能够可靠地加以计量
4. **源自合同性权利或其他法定权利**：或虽非源自合同性权利但能够从企业中分离或划分出来
5. **可分离或划分**：能够从企业中分离或划分出来，能单独或与相关合同、资产一起用于出售、转移、授予许可、租赁或交换
6. **无实物形态**：符合无形资产的非货币性、非流动性的基本特征

**路径二：确认为"存货"**

适用于企业在日常活动中持有、以备出售的数据资源产品。确认条件遵循存货准则的一般要求：

- 企业在日常活动中持有以备出售（如数据API服务、数据报告、清洗后的数据集）
- 持有目的是在正常经营过程中出售，而非长期自用
- 包括企业加工生成后准备出售的数据产品

**教学提示**：两条路径的区分本质上是对数据资源"业务用途"的分类——自用/服务型走无形资产路径，出售型走存货路径。这一分类直接决定了后续的计量规则和列示方式。

### 二、初始计量：成本范围

**外购数据资源的成本**包括购买价款、相关税费以及直接归属于使该项资产达到预定用途所发生的其他支出（如数据清洗、标注、整合等加工费用）。这与外购无形资产或存货的成本构成基本一致。

**自行开发数据资源的成本**：遵循现有无形资产准则中"研究阶段"与"开发阶段"的划分方法：

- **研究阶段**的支出：全部费用化，计入当期损益。数据资源的"研究阶段"包括数据需求调研、可行性分析、数据采集方案设计、数据质量评估与验证等前期探索性工作。
- **开发阶段**的支出：同时满足五个条件时可资本化——技术上可行、有完成并使用或出售的意图、能证明产生经济利益的方式、有足够资源支持完成、归属于该资产的开发支出能可靠计量。"开发阶段"包括数据清洗与标准化、数据标注与标签化、数据模型训练、数据产品开发、数据平台搭建等实质性开发工作。

**实务难点**：研究阶段与开发阶段的划分边界在实践中极为模糊。对于数据资源而言，从"探索性分析"到"实质性开发"的过渡往往是一个渐进过程，不像传统研发项目有明确的里程碑节点。企业需要建立清晰的内部管理制度来界定分界点。

### 三、后续计量：摊销与减值

**无形资产路径的后续计量**：

- **使用寿命确定的**：在预计使用寿命内采用系统合理的方法进行摊销（通常为直线法）。摊销金额根据数据资源的预期消耗方式确定——如果数据资源的使用价值随时间快速衰减（如实时行情数据），可能需要采用加速摊销法。
- **使用寿命不确定的**：不进行摊销，但应当在每个会计期间进行减值测试。数据资源的使用寿命评估需要考虑数据时效性、技术迭代速度、市场竞争格局等因素。
- **减值**：当有迹象表明数据资源可能发生减值时（如数据时效性严重下降、技术路线被替代、市场价值大幅缩水），应进行减值测试，减值损失计入当期损益且不可转回。

**存货路径的后续计量**：

- 按照成本与可变现净值孰低计量
- 资产负债表日，如果可变现净值低于成本，应计提存货跌价准备
- 可变现净值的确定需考虑数据产品的市场售价、销售费用及相关税费

### 四、列示与披露

**资产负债表列示**：

- 确认为无形资产的数据资源：在"无形资产"项目下列示，企业可在报表附注中单独披露数据资源的明细（首次明确允许）
- 确认为存货的数据资源：在"存货"项目下列示

**附注披露要求（强制性）**：

- 数据资源的类别（无形资产/存货）及各类的期初和期末账面价值、本期增减变动情况
- 确认为无形资产的数据资源的摊销方法、使用寿命及减值情况
- 确认为存货的数据资源的跌价准备计提方法
- 数据资源的取得方式和成本构成
- 对数据资源使用寿命及摊销方法进行估计的信息

**自愿性披露（鼓励）**：

- 数据资源的评估价值及公允价值（如有）
- 与数据资源相关的重大交易或事项
- 数据资源的应用场景和对企业财务状况的影响
- 数据资源的权利瑕疵或受限情况
- 其他有助于信息使用者决策的信息

### 五、企业实务影响

本规定的影响范围极为广泛，但对不同行业的影响程度和方式存在显著差异：

- **互联网平台企业**：拥有海量用户行为数据、交易数据，数据资源价值巨大但此前无法体现在财务报表中。入表后可显著改善资产负债表结构，但同时面临数据确权和成本归集的巨大挑战。
- **数据交易/数据服务商**：以数据产品为核心业务的企业，数据资源本身就是其核心资产，入表直接影响其资产估值和融资能力。
- **金融科技企业**：积累大量风控数据、信用评估数据，这些数据资源入表后有助于更真实地反映企业的核心竞争力和资产质量。
- **智能制造/工业互联网企业**：拥有大量生产过程数据、设备运行数据，入表推动企业将数据从"生产副产物"重新认知为"核心资产"。
- **传统行业企业**：如零售企业积累的消费者画像数据、物流企业的运输数据等，此前往往被忽视，入表提供了重新审视和盘活数据资产的制度工具。

## 制度创新

| 创新点 | 内容 | 全球意义 |
|--------|------|----------|
| **率先明确数据资产入表路径** | 将数据资源嵌入无形资产和存货框架，提供可操作的确认和计量规则 | 填补了国际会计准则在数据资产领域的制度空白，为全球提供了可参考的先行方案 |
| **区分"无形资产"与"存货"双路径** | 根据业务用途（自用vs出售）分类处理，兼顾不同商业模式 | 体现了对数据资源多样化应用场景的深刻理解，超越了单一资产类别的局限 |
| **建立分层披露机制** | 强制性披露（确保信息底线）+ 自愿性披露（鼓励信息丰富） | 在会计信息可靠性与相关性之间取得平衡，为投资者提供决策有用的增量信息 |
| **衔接"数据二十条"的顶层设计** | 将"数据资产入表"从政策口号转化为可执行的会计规则 | 展示了中国的政策执行力——从顶层规划到制度落地的完整闭环，是政策实施的标杆案例 |

## 全球视角：数据资产化的国际探索

| 国家/地区 | 做法 | 进展 |
|-----------|------|------|
| **中国** | 财政部发布专门会计规定，明确入表路径 | 2024年1月正式施行，全球首个数据资源会计规范 |
| **IASB（国际会计准则理事会）** | 尚未发布专门准则，但已将数据资产相关议题列入研究议程 | 目前处于讨论和研究阶段，关注点包括数据资产的定义、计量和披露 |
| **FASB（美国财务会计准则委员会）** | 未专门规定数据资产会计处理 | 企业通过现有无形资产、商誉或研发支出等准则框架间接处理 |
| **欧盟** | 在《数字运营韧性法案》等法规中涉及数据管理要求，但未制定专门的会计规范 | 侧重于数据治理与安全，会计层面的规范尚在探索中 |
| **新加坡** | 作为亚洲金融中心，关注数据资产的估值和交易，但未出台专门会计准则 | 侧重数据交易市场建设，会计层面跟随国际准则 |

**国际比较的启示**：中国在这项制度上的先行优势是显著的。IASB和FASB的保守态度源于数据资产本身的不确定性（确权难、估值难、时效性强），而中国通过"嵌入现有框架"的务实策略有效降低了制度创新风险。中国的先行经验很可能影响未来国际准则的制定方向。

## 文件间关系

- **与《数据二十条》(P085)**：本规定是"数据二十条"第(十二)条"探索数据资产入表新模式"的直接制度落地，将顶层政策转化为可操作的会计规则。
- **与P012（要素市场化配置）**：本规定为"数据作为生产要素"提供了会计核算的制度基础——要素只有被确认为资产、被计量、被报告，才能真正实现市场化配置。
- **与数据交易场所制度**：数据资源的会计确认和计量为数据交易提供了价值评估和定价的会计基础，是数据交易市场健康发展的底层支撑。
- **与《数据安全法》《个人信息保护法》**：数据资源入表的前提是"合法拥有或控制"，合规性审查成为入表的前置条件，两部法律构成了入表的合规底线。

## 教学要点

**一、课堂讨论**

> "数据资源入表——对企业来说是'资产重估'的机会还是'合规负担'？没有入表前的数据资产怎么处理？"

引导学生从多个角度展开讨论：
- **机会视角**：改善资产负债表、提升企业估值、为数据融资提供基础、推动企业重视数据管理
- **负担视角**：确权难度大、成本归集复杂、减值测试具有主观性、披露可能暴露商业秘密
- **历史衔接**：2024年1月1日施行前的数据资源投入能否追溯调整？根据衔接规定，采用未来适用法，不进行追溯调整

**二、核心概念**

| 概念 | 定义 | 在本规定中的体现 |
|------|------|------------------|
| 无形资产 | 企业拥有或控制的、没有实物形态的、可辨认的非货币性资产 | 数据资源作为无形资产入表的核心路径 |
| 存货 | 企业在日常活动中持有以备出售的产成品或商品 | 数据产品（API服务、数据集等）的入表路径 |
| 资本化 | 将支出计入资产成本而非当期费用的会计处理 | 自行开发数据资源"开发阶段"支出的资本化处理 |
| 摊销 | 在使用寿命内将无形资产成本系统分摊计入费用的过程 | 数据资源无形资产的后续计量方法 |
| 减值 | 当资产可收回金额低于账面价值时确认的损失 | 数据资源因时效性下降或技术替代导致的减值 |
| 可变现净值 | 估计售价减去至完工时估计的成本和估计的销售费用 | 数据产品存货减值判断的核心指标 |

**三、实务难点**

1. **数据确权难题**：中国法律尚未完全建立数据财产权体系，"合法拥有或控制"在实务中如何判断？数据来源是否合规？加工后的衍生数据是否产生新的财产权？这些问题直接关系到能否满足入表的第一项条件。

2. **价值评估难题**：数据资产的非标准化特征使其价值评估极为困难。不同用途、不同时效、不同场景下，同一数据集的价值可能差异巨大。减值测试中的"可收回金额"如何确定，是实务中的核心挑战。

3. **成本归集难题**：传统企业的成本核算体系通常不专门追踪数据相关支出。数据采集、清洗、加工、存储的成本往往分散在IT费用、研发费用、运营费用等多个科目中，准确归集到具体数据资源上的成本难度极大。

**四、学生思考题**

1. 某互联网公司投入5000万元采集和标注用户行为数据用于推荐算法优化，应如何判断其中哪些支出可以资本化、哪些必须费用化？分界标准是什么？
2. 数据资源的使用寿命应如何确定？哪些因素会导致使用寿命不确定？试举三个不同行业的例子加以说明。
3. 如果一家企业将其数据资源同时用于内部业务优化（无形资产路径）和对外销售（存货路径），应如何进行分类和会计处理？
4. 从会计信息质量的角度分析：数据资源入表的强制披露与自愿披露机制设计是否合理？是否存在信息过载或披露不足的风险？
5. 对比中国与IASB在数据资产会计规范上的不同策略（嵌入现有框架 vs 制定新准则），分析各自的利弊。

**五、延伸阅读**

1. 财政部会计司有关负责人就《企业数据资源相关会计处理暂行规定》答记者问（理解立法意图和实务操作指引）
2. 《关于构建数据基础制度更好发挥数据要素作用的意见》（"数据二十条"，P085）第(十二)条
